# Brand Identity (Protocol)

## When to use this skill

- User asks for "brand guidelines"
- User mentions "consistency" or "tone"
- Triggered by context requiring brand alignment

## Core Protocols

- [Sparks Design System & Doctrine](../../../../../personas/sparks_design_system.md)

## Workflow

1.  **Check Guidelines**: Review `sparks_design_system.md` in `personas/` for "Safe Zones" and "Novelty Rules".
2.  **Verify Assets**: Ensure colors match the "Cosmic System" (neutral grays, cosmic blue accents).
3.  **Enforce Voice**: Ensure communication is "Calm, welcoming, quietly confident".
4.  **Reject Novelty**: If a design choice is purely ornamental, flag it.
